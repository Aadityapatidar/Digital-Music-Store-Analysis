# Digital-Music-Store-Analysis

# Introduction of data
Each row represents a relationship between a playlist and a track.
playlist_id identifies the playlist.
track_id identifies the track included in that playlist.
A single playlist can include multiple tracks.
A single track can appear in multiple playlists.
The dataset supports analysis of playlist structure, track distribution, and music recommendation patterns.


# Conclusion

This project analyzes a digital music store dataset using SQL to uncover insights about customers, artists, genres, and sales performance. It connects playlists, tracks, invoices, and customer data to study music purchasing and listening patterns. Using joins, subqueries, window functions, and CTEs, the project identifies the best customers, top-selling artists, most popular genres by country, and countries with the highest sales. It also finds Rock music listeners and tracks longer than average. The analysis supports business decisions such as targeted marketing, customer segmentation, regional trend identification, and revenue optimization, demonstrating the practical use of SQL for real-world data analytics.
